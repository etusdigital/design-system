BRoundButton Accessibility Testing Suite

This page tests the accessibility implementation of the BRoundButton component according to WCAG 2.1 AA guidelines.

Testing Instructions

  • Use Tab to navigate between buttons
  • Press Enter or Space to activate buttons
  • Enable screen reader (VoiceOver on Mac: Cmd + F5)
  • Check focus indicators are visible
  • Verify announcements in the results section

Icon-Only Button Labeling

Test: Icon-only buttons must have accessible labels for screen readers.
Expected: Screen readers announce meaningful labels, not just "button".

Keyboard Navigation

Test: Buttons must be keyboard accessible with Enter and Space keys.
Expected: Tab focuses buttons, Enter/Space activates them.

Toggle Button States

Test: Toggle buttons must announce state changes using aria-pressed.
Expected: Screen readers announce "pressed" or "not pressed" states.

Loading States

Test: Loading buttons must use aria-busy and announce state changes.
Expected: Screen readers announce loading state and completion.

Disabled States

Test: Disabled buttons must be focusable but not activatable for screen readers.
Expected: Tab can focus disabled buttons, they announce as disabled.
No printer is currently available

Touch Target Size

Test: All buttons must meet minimum 44px touch target size.
Expected: All buttons are at least 44x44 pixels for touch accessibility.
