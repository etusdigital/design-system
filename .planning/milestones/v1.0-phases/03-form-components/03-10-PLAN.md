**Slider.tsx changes:**

1. Add import: `import { Tooltip } from '../Tooltip';`

2. Replace the raw div tooltip inside the cursor mapping (approx lines 429-433). Currently each cursor div contains:
```tsx
{showTooltip && (
  <div className={styles.tooltip}>
    {getTooltipText(modelArray[index])}
  </div>
)}
```

Instead, wrap the entire cursor `<div>` element with the DS Tooltip component. The cursor div (with className styles.cursor) becomes the Tooltip's child (trigger). The Tooltip shows on hover automatically.

Replace the cursor rendering to:
```tsx
{showTooltip ? (
  <Tooltip
    labelValue={getTooltipText(modelArray[index])}
    position={vertical ? 'right' : 'top'}
  >
    <div
      key={index}
      className={clsx(styles.cursor, draggingIndex === index && styles.dragging)}
      style={getCursorStyle(index)}
      onMouseDown={(e) => {
        e.stopPropagation();
        startDragging(index);
      }}
      onTouchStart={(e) => {
        e.stopPropagation();
        startDragging(index);
      }}
    />
  </Tooltip>
) : (
  <div
    key={index}
    className={clsx(styles.cursor, draggingIndex === index && styles.dragging)}
    style={getCursorStyle(index)}
    onMouseDown={(e) => {
      e.stopPropagation();
      startDragging(index);
    }}
    onTouchStart={(e) => {
      e.stopPropagation();
      startDragging(index);
    }}
  />
)}
```

Key: `position={vertical ? 'right' : 'top'}` — this automatically handles UAT gap 6 (vertical tooltip on right side).

3. Simplify step markers (approx lines 437-447). Remove the label span entirely. The step markers should render as simple thin ticks:
```tsx
{steps && steps.map((step, i) => (
  <div
    key={i}
    className={clsx(styles.step)}
    style={getStepStyle(step)}
  >
    <div className={clsx(styles.stepMarker, isStepActive(step) && styles.stepMarkerActive)} />
  </div>
))}
```

Remove the `{step.label && <span className={styles.stepLabel}>{step.label}</span>}` line.

**Slider.module.css changes:**

1. Remove the entire `.tooltip` class block (lines 82-86). The DS Tooltip handles its own styling.

2. Simplify `.stepMarker` — change the height from oversized to a thin tick:
```css
.stepMarker {
  @apply h-[0.75em] w-[0.125em] bg-primary-surface-highlight rounded-full;
}
```
This creates a small thin tick mark instead of the tall 1.875em marker.

3. Remove the `.stepLabel` class entirely — step labels are no longer rendered.

4. Keep `.step`, `.stepActive`, `.stepMarkerActive` classes as they are.
