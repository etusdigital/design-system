<task type="auto">
  <name>Task 2: Fix ToggleGroup border collapse and rounded corners</name>
  <files>src/components/ToggleGroup/ToggleGroup.module.css, src/components/Toggle/Toggle.module.css</files>
  <action>
**Toggle.module.css changes:**
The `.toggle` class currently applies `border border-current` on all 4 sides. This is fine as the base — ToggleGroup CSS will override for connected layout.

Add a new class `.inGroup` that applies `rounded-none` so ToggleGroup can reset rounding and selectively restore it:
```
.inGroup {
  @apply rounded-none;
}
```

**ToggleGroup.module.css changes — horizontal default:**
Replace the current `.default > *` rules with adjacent-sibling border collapse:

```css
/* Default type: connected pill layout */
.default > * {
  @apply rounded-none;
}

/* Remove left border on all items after the first (adjacent sibling) */
.default:not(.vertical) > * + * {
  @apply border-l-0;
}

/* Rounded corners on first and last child */
.default:not(.vertical) > *:first-child {
  @apply rounded-l-sm;
}

.default:not(.vertical) > *:last-child {
  @apply rounded-r-sm;
}
```

**ToggleGroup.module.css changes — vertical default:**
```css
.default.vertical > * + * {
  @apply border-t-0;
}

.default.vertical > *:first-child {
  @apply rounded-t-sm;
}

.default.vertical > *:last-child {
  @apply rounded-b-sm;
}
```

Remove ALL existing `.default > *`, `.default > *:last-child`, `.default:not(.vertical) > *:first-child`, `.default:not(.vertical) > *:last-child`, `.default.vertical > *`, `.default.vertical > *:last-child`, `.default.vertical > *:first-child`, `.default.vertical > *:last-child` rules and replace with the above.

The key fix: use `> * + *` (adjacent sibling) to remove only the shared border between items, instead of removing border-right from all items and re-adding it on the last.
  </action>
