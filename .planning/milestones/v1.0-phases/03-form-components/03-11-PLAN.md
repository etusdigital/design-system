In Slider.tsx, find the isStepActive function (around line 317-325). Currently:

```typescript
const isStepActive = (step: { label: string; value: number }): boolean => {
  const pct = max ? step.value / max : step.value;
  const pcts = isRange
    ? [toPercentage(modelArray[0]), toPercentage(modelArray[1])]
    : [toPercentage(modelArray[0])];
  const minPct = Math.min(...pcts);
  const maxPct = Math.max(...pcts);
  return pct >= minPct && pct <= maxPct;
};
```

The bug: for single-value sliders, `pcts` has one element so `minPct === maxPct === cursorPosition`. The fill bar goes from 0 to cursor, but the check `pct >= minPct` excludes all steps before the cursor.

Fix: for single-value sliders, use 0 as the lower bound (fill starts at 0). For range sliders, keep using minPct (fill is between the two thumbs).

Replace with:

```typescript
const isStepActive = (step: { label: string; value: number }): boolean => {
  const pct = max ? step.value / max : step.value;
  if (isRange) {
    const minPct = Math.min(toPercentage(modelArray[0]), toPercentage(modelArray[1]));
    const maxPct = Math.max(toPercentage(modelArray[0]), toPercentage(modelArray[1]));
    return pct >= minPct && pct <= maxPct;
  }
  return pct >= 0 && pct <= toPercentage(modelArray[0]);
};
```

This ensures:
- Single slider: steps from 0 to cursor position are active (matching the fill bar)
- Range slider: steps between the two thumbs are active (matching the fill bar)
