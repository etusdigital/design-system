**Input.module.css — add border-0:**
In the `.inputContent` class, add `border-0` to the @apply directive. This neutralizes the 1px border that @tailwindcss/forms plugin injects on all input elements. The current line:
```
@apply flex-1 bg-transparent outline-none text-sm text-neutral-foreground-high placeholder:text-neutral-foreground-low min-w-0;
```
becomes:
```
@apply flex-1 bg-transparent outline-none border-0 text-sm text-neutral-foreground-high placeholder:text-neutral-foreground-low min-w-0;
```

**Input.tsx — fix number arrows centering:**
On approximately line 237, remove `mt-xxs` from the number arrows wrapper className. The current code:
```tsx
<div className={clsx(styles.numberArrows, 'ml-xxs mt-xxs')}>
```
becomes:
```tsx
<div className={clsx(styles.numberArrows, 'ml-xxs')}>
```
The flex container with `items-center` in `.numberArrows` already handles vertical centering — `mt-xxs` was pushing the arrows down.

**Input.tsx — fix mask guard to allow domain/url types:**
On approximately line 118, the mask is only applied when `type === 'text'`. Domain and URL are not mask-based inputs — they are validation-based. Do NOT change the mask guard. Instead, fix the validation feedback:

**Input.tsx — add auto error message for domain/url on blur:**
The Input component already has blur validation logic for domain/url types using `isValidDomain` and `isValidUrl`. The problem is that `errorMessage` is only shown when explicitly passed as a prop. Add internal error state:

1. Add a `useState` for internal validation error: `const [validationError, setValidationError] = useState<string>('')`
2. In the blur handler for domain/url types, when validation fails, call `setValidationError('Invalid domain')` or `setValidationError('Invalid URL')` respectively. When validation passes, call `setValidationError('')`.
3. In the error message render section (approx line 253), change:
   ```tsx
   {errorMessage && showError && (
   ```
   to:
   ```tsx
   {(errorMessage || validationError) && showError && (
     <p className={styles.errorMessage}>{errorMessage || validationError}</p>
   )}
   ```

Also ensure `showError` defaults to true or is set appropriately when there's a validationError.

**utils/index.ts — handle domain/url in applyMask:**
The `applyMask` function has `domain` and `url` in its type signature but no switch case. Since these types don't need character masking (they're validated on blur, not formatted on input), add explicit no-op cases so the function doesn't silently fall through:
```typescript
case "domain":
case "url":
case "email":
  // No character masking — validated on blur
  break;
```
Add these cases before the closing of the switch statement.
