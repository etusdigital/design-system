Replace the Dialog.tsx stub with the full implementation. The Vue source is Dialog.vue (105 lines with scoped CSS).

**Dialog.tsx implementation:**

```typescript
import { useRef } from 'react';
import clsx from 'clsx';
import { Overlay } from '../../utils/components/Overlay';
import { useControllable } from '../../hooks/useControllable';
import { useTransition } from '../../hooks/useTransition';
import './Dialog.css';

export interface DialogProps {
  value?: boolean;
  defaultValue?: boolean;
  onChange?: (value: boolean) => void;
  width?: string;
  height?: string;
  noOutsideClose?: boolean;
  children?: React.ReactNode;
  className?: string;
}
```

- `useControllable<boolean>({ value, defaultValue, onChange })` for open/close state
- `useTransition(isOpen ?? false, { duration: 500 })` for bounce animation timing
- `dialogRef = useRef<HTMLDivElement>(null)` for shake animation classList manipulation
- `handleOverlayClick()`: if `noOutsideClose`, add `'no-outside-close-warning'` class to dialogRef, `setTimeout(() => remove class, 100)`. Otherwise call `setOpen(false)`.
- Render: `<Overlay value={isOpen} zIndex={1002} onClick={handleOverlayClick}>` wrapping `{isMounted && <div ref={dialogRef} className={clsx('dialog', isActive && 'active', className)} style={{ width, height }}>{children}</div>}`
- Default props: `width = 'fit-content'`, `height = 'fit-content'`, `noOutsideClose = false`

**Dialog.css implementation** (plain CSS, NOT module — consistent with Overlay.css pattern):

```css
@reference "../../assets/main.css";

.dialog {
  @apply fixed z-[1003] top-[50%] left-[50%] bg-neutral-surface-default rounded-base border-xxs border-neutral-default;
  transform: translate(-50%, -50%) scale(0);
  opacity: 0;
  max-width: calc(100% - var(--spacing-xl));
  max-height: calc(100% - var(--spacing-xl));
  transition: transform 0.5s ease, opacity 0.5s ease;
}

.dialog.active {
  transform: translate(-50%, -50%) scale(1);
  opacity: 1;
}

.no-outside-close-warning {
  animation: bounce-warning 0.5s ease;
}

@keyframes bounce-warning {
  0% {
    transform: translate(-50%, -50%) scale(1.1);
  }
  100% {
    transform: translate(-50%, -50%) scale(1);
  }
}
```

Note: The Vue version uses `@keyframes bounce-in` for entry, but in React the `useTransition` + CSS transition approach handles entry/exit via `transform`+`opacity` transition triggered by `.active` class. The `bounce-warning` keyframe is only for the noOutsideClose shake.
