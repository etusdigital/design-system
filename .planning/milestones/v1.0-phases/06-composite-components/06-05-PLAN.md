Port Filter.vue (331 lines) to React TSX. Filter is a multi-level checkbox filter that uses SelectContainer infrastructure.

Props interface `FilterProps`:
- `value?: Record<string, any[]>` — selected values per category (controlled)
- `defaultValue?: Record<string, any[]>` — initial selections (uncontrolled)
- `onChange?: (value: Record<string, any[]>) => void` — selection change
- `onApply?: (value: Record<string, any[]>) => void` — apply callback
- `options: Array<{ label: string; value: string; options: any[] }>` — categories with sub-options
- `labelKey?: string` — default `'label'`
- `valueKey?: string` — default `'value'`
- `disabled?: boolean` — default `false`
- `labelValue?: string` — trigger label, default `''`
- `clearLabel?: string` — clear button text, default `'Clear selection'`
- `applyLabel?: string` — apply button text, default `'Apply filters'`
- `statusLabel?: string` — status text, default `'Status'`
- `isError?: boolean` — default `false`
- `errorMessage?: string` — default `''`
- `className?: string`

Implementation:
- `useControllable<Record<string, any[]>>({ value, defaultValue: {}, onChange })` for per-category selections
- `useState<Record<string, boolean>>({})` for category expanded states
- Category expand/collapse: max-height CSS transition (NOT useTransition — per CONTEXT.md locked decision). Toggle expanded state per category key.
- Each category header: click toggles expand state, shows `expand_more` icon rotating 180deg
- Sub-options: `Checkbox` + label text at `text-sm`. Checkbox `value` = whether sub-option is in category's selected array. `onChange` toggles in/out.
- Actions row: Clear Button (neutral, calls `setModel({})`) + Apply Button (primary, calls `onApply?.(model)`)
- Status display: count of total selected items across all categories
- Use `isObject` from `src/utils/index.ts` for option value extraction
