Read the Vue source `src/components/Stepper/Stepper.vue` to understand the target design. The React version has several issues:

**Stepper.tsx fixes:**

1. **Fix biggerStepSelected tracking:** Should be a simple number tracking the maximum step index the user has reached (not an array or boolean). Initialize as `Math.max(0, value ?? 0)`. Update on every step change: `setBiggerStepSelected(prev => Math.max(prev, newValue))`.

2. **Fix connector coloring logic:** The `getConnectorState` function must return:
   - `'past'` (green) when `index < biggerStepSelected` — the connector BETWEEN step[index] and step[index+1] is green only if step[index+1] has been visited
   - `'future'` (gray) when `index >= biggerStepSelected`

   Currently it colors the connector AT the current index green (wrong — should color connectors BEFORE current).

3. **Add background ring element for active step:** Each step circle should have a layered design:
   - Outer: background ring div (larger circle, semi-transparent primary color)
   - Inner: foreground circle with icon/number
   - Active step: ring visible with split coloring (past side = primary, future side = gray)
   - Past steps: filled primary circle with checkmark icon
   - Future steps: gray bordered circle

4. **Add scale-[1.2] on active step:** The step item container for the active step should have `transform: scale(1.2)` applied.

5. **Differentiate circle styles:**
   - Past: `bg-primary-interaction-default text-neutral-foreground-negative` (green bg, white text/icon)
   - Active: `bg-primary-interaction-default text-neutral-foreground-negative` with ring
   - Future: `bg-neutral-surface-default border-xxs border-neutral-border-default text-neutral-foreground-low`

**Stepper.module.css fixes:**

1. Change `.stepper` from `items-start` to `items-center` so connectors align with circle centers.

2. Add `.ring` class for the background ring element:
   ```css
   .ring {
     @apply absolute inset-[-4px] rounded-full;
     background: conic-gradient(
       var(--primary-surface-highlight) 0deg,
       var(--neutral-surface-highlight) 0deg
     );
   }
   ```
   The conic-gradient angle will be set dynamically based on the split point.

3. Differentiate `.past`, `.active`, `.future` circle classes with distinct background and border colors.

4. Add `.activeItem` with `transform: scale(1.2)` and `transition: transform 200ms ease`.

5. Ensure connector `height: 2px` and `top: 50%` with `transform: translateY(-50%)` for vertical centering within the flex row.

<task type="auto">
  <name>Task 2: Fix Carousel section rendering and add disabled/circular props</name>
  <files>src/components/Carousel/Carousel.tsx, src/components/Carousel/Carousel.module.css</files>
  <action>
Read the Vue source for Carousel to understand the section-based rendering model.

**Carousel.tsx fixes:**

1. **Add disabled and circular props** to CarouselProps:
   ```typescript
   disabled?: boolean;
   circular?: boolean;
   ```

2. **Fix section rendering:** The Vue version groups visible items into "sections" (pages). Each section is a flex container showing N items at once. The React version currently flatly renders all items without section wrappers.

   Fix: In the `sections` computation, each section should be an array of items. Then in the render:
   ```tsx
   <div className={styles.track} style={{ transform: `translateX(-${currentIndex * 100}%)` }}>
     {sections.map((section, sectionIndex) => (
       <div key={sectionIndex} className={styles.section}>
         {section.map((item, itemIndex) => (
           <div key={itemIndex} className={styles.slide}>
             {item}
           </div>
         ))}
       </div>
     ))}
   </div>
   ```

3. **Apply overflow constraints:** The overflow container div needs fixed width/height from `calculateContentStyle`. Add computed style:
   ```tsx
   style={{ width: '100%', overflow: 'hidden' }}
   ```

4. **Wire disabled prop:** When `disabled={true}`:
   - Arrow buttons get `disabled` attribute and reduced opacity
   - Dot navigation still shows but clicks are ignored
   - Autoplay is paused

5. **Wire circular prop:** When `circular={true}`:
   - Navigating past the last section wraps to first
   - Navigating before first section wraps to last
   - Arrow buttons are never disabled

6. **Fix calculateContentStyle:** Should measure actual child DOM widths using a ref, not rely on hardcoded values. Use `useEffect` + `useRef` on the track container to measure children after mount.

**Carousel.module.css fixes:**

1. Add `.track` with `display: flex; width: max-content; transition: transform 300ms ease;`
2. Add `.section` with `display: flex; flex-shrink: 0;` and width matching the viewport
3. Ensure `.slide` has appropriate flex sizing
