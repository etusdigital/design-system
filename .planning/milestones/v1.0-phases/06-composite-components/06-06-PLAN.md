Port Carousel.vue (324 lines) to React TSX. Carousel slides items with CSS translateX/translateY and supports autoplay.

Props interface `CarouselProps`:
- `value?: number` — active slide index (controlled)
- `defaultValue?: number` — initial slide (uncontrolled)
- `onChange?: (value: number) => void` — slide change callback
- `options: any[]` — items to display
- `visible?: number` — number of visible items per page, default `1`
- `vertical?: boolean` — vertical sliding, default `false`
- `autoplay?: boolean` — auto-advance slides, default `false`
- `autoplayInterval?: number` — interval in ms, default `3000`
- `showArrows?: boolean` — show prev/next arrows, default `true`
- `children?: (option: any, index: number) => React.ReactNode` — children-as-function for option rendering (per CONTEXT.md locked decision)
- `className?: string`

Implementation:
- `useControllable<number>({ value, defaultValue: 0, onChange })` for active index
- `carouselRef = useRef<HTMLDivElement>(null)` for container measurements
- `const [contentStyle, setContentStyle] = useState({})` for slide position

**useLayoutEffect** (replaces Vue nextTick):
```typescript
useLayoutEffect(() => {
  calculateContentStyle();
}, [model, options.length, visible]);
```

**calculateContentStyle** function:
- Read container dimensions via `carouselRef.current.offsetWidth` / `offsetHeight`
- Read gap from CSS variable: `getComputedStyle(document.documentElement).getPropertyValue('--spacing-xs')` — use `document.documentElement` NOT the container element (per RESEARCH.md pitfall 7)
- Calculate item width: `(containerWidth - gap * (visible - 1)) / visible`
- Calculate translate offset: `model * (itemWidth + gap)`
- Set transform: `translateX(-${offset}px)` or `translateY(-${offset}px)` for vertical

**Autoplay** via `setInterval` in `useEffect`:
```typescript
useEffect(() => {
  if (!autoplay) return;
  const totalPages = Math.ceil(options.length / visible);
  const interval = setInterval(() => {
    setModel(prev => (prev + 1) % totalPages);
  }, autoplayInterval);
  return () => clearInterval(interval);
}, [autoplay, autoplayInterval, options.length, visible]);
```

**Navigation arrows**: Material Symbols `chevron_left`/`chevron_right` (horizontal) or `expand_less`/`expand_more` (vertical). `aria-label="Previous slide"` and `aria-label="Next slide"`.

**Children-as-function** rendering:
```typescript
{sections.map((section, sectionIndex) => (
  section.map((option, optionIndex) => (
    <div key={optionIndex} inert={sectionIndex !== model ? '' : undefined}>
      {children?.(option, sectionIndex * visible + optionIndex)}
    </div>
  ))
))}
```

Group options into sections of `visible` length.

Port RoundMenu.vue (88 lines) to React TSX. RoundMenu positions items in a circle using trigonometric calculations.

Props interface `RoundMenuProps`:
- `options: Array<{ icon?: string; label?: string; onClick?: () => void; [key: string]: any }>` — menu items
- `iconKey?: string` — key for icon, default `'icon'`
- `labelKey?: string` — key for label, default `'label'`
- `radius?: number` — distance from center in px, default `80`
- `className?: string`

Implementation:
- `useState(false)` for isExpanded
- Center trigger button: toggles isExpanded. When expanded: `bg-primary-interaction-default text-neutral-foreground-negative`
- `calculatePosition(index, total)`: angle = `(2 * Math.PI * index) / total - Math.PI / 2` (start from top). x = `Math.cos(angle) * radius`. y = `Math.sin(angle) * radius`. Return `{ transform: \`translate3d(${x}px, ${y}px, 0)\` }`.
- Each item: `position: absolute`, centered on trigger, with calculated transform when expanded. Use `transition: transform 0.3s ease`.
- When collapsed: all items at center (`transform: translate3d(0, 0, 0) scale(0)`)
- When expanded: items at calculated position (`transform: translate3d(x, y, 0) scale(1)`)
- Item buttons: `rounded-full`, `bg-neutral-surface-default border-xxs border-neutral-default shadow-neutral-default`
- Icon inside each item using `Icon` component
