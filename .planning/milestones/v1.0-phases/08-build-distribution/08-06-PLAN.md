**Root cause:** `vite-plugin-dts@4.5.4` reads `tsconfig.lib.json` which has `noEmit: true` and `verbatimModuleSyntax: true`. The combination causes the plugin to produce empty `export {}` stubs because TypeScript's internal API treats `noEmit` as "don't produce declarations" and `verbatimModuleSyntax` causes barrel re-exports (`export { X } from './X'`) to be treated as type-only in certain configurations.

**Fix — Step 1: Create `tsconfig.dts.json`** at project root:

```json
{
  "extends": "./tsconfig.lib.json",
  "compilerOptions": {
    "noEmit": false,
    "declaration": true,
    "declarationMap": false,
    "emitDeclarationOnly": true,
    "verbatimModuleSyntax": false,
    "erasableSyntaxOnly": false
  }
}
```

This extends tsconfig.lib.json (inheriting include/exclude, paths, strict settings) but overrides:
- `noEmit: false` + `declaration: true` + `emitDeclarationOnly: true` — tells TS to produce .d.ts files
- `verbatimModuleSyntax: false` — ensures barrel re-exports are not stripped as type-only
- `erasableSyntaxOnly: false` — required when verbatimModuleSyntax is false

**Fix — Step 2: Update `dts()` plugin config in `vite.config.ts`:**

Add a new section after the existing checks that verifies .d.ts files have real content. Add these checks:

```javascript
// === Type Declaration Checks ===
console.log('\n--- Type Declaration Checks ---');
let dtsErrors = 0;

// Check main.d.ts has real re-exports
const mainDts = fs.readFileSync('./lib/main.d.ts', 'utf-8');
if (!mainDts.includes('export')) {
  console.error('FAIL: lib/main.d.ts has no exports');
  dtsErrors++;
} else {
  console.log('PASS: lib/main.d.ts has exports');
}

// Spot-check per-component .d.ts files have real content (not just "export {}")
const spotCheckComponents = ['Button', 'Input', 'Select', 'Dialog', 'Table'];
for (const comp of spotCheckComponents) {
  const dtsPath = `./lib/components/${comp}/index.d.ts`;
  if (!fs.existsSync(dtsPath)) {
    console.error(`FAIL: ${dtsPath} does not exist`);
    dtsErrors++;
    continue;
  }
  const content = fs.readFileSync(dtsPath, 'utf-8');
  if (content.trim() === 'export {}' || content.length < 20) {
    console.error(`FAIL: ${dtsPath} is empty stub (${content.length} bytes)`);
    dtsErrors++;
  } else if (!content.includes(comp)) {
    console.error(`FAIL: ${dtsPath} does not contain '${comp}'`);
    dtsErrors++;
  } else {
    console.log(`PASS: ${dtsPath} has real types (${content.length} bytes)`);
  }
}

// Check hook .d.ts
const hookDtsPath = './lib/hooks/useControllable.d.ts';
if (fs.existsSync(hookDtsPath)) {
  const hookContent = fs.readFileSync(hookDtsPath, 'utf-8');
  if (hookContent.trim() === 'export {}' || hookContent.length < 20) {
    console.error(`FAIL: ${hookDtsPath} is empty stub`);
    dtsErrors++;
  } else {
    console.log(`PASS: ${hookDtsPath} has real types (${hookContent.length} bytes)`);
  }
} else {
  console.error(`FAIL: ${hookDtsPath} does not exist`);
  dtsErrors++;
}

if (dtsErrors > 0) {
  console.error(`\n${dtsErrors} type declaration error(s) found`);
  process.exitCode = 1;
} else {
  console.log('\nAll type declaration checks passed');
}
```

Integrate this into the existing script structure, preserving all existing checks. The existing `errors` counter and the new `dtsErrors` counter should both contribute to the final exit code.
