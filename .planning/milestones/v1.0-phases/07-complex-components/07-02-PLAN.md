**Architecture (3 components, all at MODULE SCOPE):**

1. **SidebarSubOption** (module scope, not exported):
   - Props: `option: SidebarOptionType`, `depth: number`
   - Has own `expanded` boolean state (useState) for showing nested children
   - Renders option label + expand icon if has children
   - Recursively renders children as SidebarSubOption when expanded
   - Click handler: if has children, toggle expand; if leaf, call parent onChange via context or prop

2. **SidebarOption** (module scope, not exported):
   - Props: `option: SidebarOptionType`
   - Renders icon + label (label hidden when sidebar collapsed)
   - Active state: check if option.value matches current value via context
   - Click: if has sub-options, expand sub-panel; if leaf, navigate

3. **Sidebar** (exported):
   - `useControllable` for value binding
   - Split options into top (option.bottom !== true) and bottom (option.bottom === true)
   - Height calculation via useEffect: `querySelector('.navbar')?.getBoundingClientRect().height` -> `calc(100vh - ${h}px)`
   - Window resize listener for height recalculation
   - `expanded` prop controls collapsed (icon-only, 64px) vs expanded mode
   - Sub-panel: when a main-rail option with children is clicked, show sub-options panel absolutely positioned to the right
   - `onBlur` with `e.relatedTarget` check to close sub-panel when focus leaves
