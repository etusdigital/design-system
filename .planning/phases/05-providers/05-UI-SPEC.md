## Spacing Scale

Declared values sourced from `src/assets/main.css` CSS variables and `tailwind.config.cjs`. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--spacing-xxs`) | Icon gaps, inline padding, gap between Confirm buttons |
| sm | 8px (`--spacing-xs`) | Toast container gap between stacked toasts, Confirm title–button gap (`mt-sm`) |
| md | 16px (`--spacing-base`) | Alert padding, Dialog internal content padding |
| lg | 24px (`--spacing-xl`) | Confirm inner padding (`p-xl`) |
| xl | 32px (`--spacing-2xl`) | Layout gaps |
| 2xl | 48px (`--spacing-3xl`) | Major section breaks |
| 3xl | 64px (`--spacing-5xl`) | Page-level spacing |

Exceptions:
- `--spacing-sm` is 12px in this system (used as `gap-sm` inside Confirm title/message column). Not a multiple of 4. Preserved as-is for Vue visual parity.
- Toast container uses `p-sm` (12px) padding — same exception applies.
- Dialog `max-width` and `max-height` are computed: `calc(100% - var(--spacing-xl))` = `calc(100% - 24px)`.

## Typography

Sourced from `tailwind.config.cjs` utility classes and `main.css` base styles. Maximum 2 weights used per component.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / Toast message | 14px (`text-sm`) | 400 (normal) | 120% (`leading-lg`) |
| Label / Toast title | 16px (`text-base`) | 700 (bold) | 105% (`leading-xs`) |
| Heading / Confirm title | 18px (`text-lg`) | 700 (bold) | 120% (`leading-lg`) |
| Caption / Confirm message | 14px (`text-sm`) | 400 (normal) | 120% (`leading-lg`) |

Notes:
- Confirm title: `font-bold text-lg text-neutral-foreground-high` — 18px/700
- Confirm message: `text-sm text-neutral-foreground-low` — 14px/400
- Alert (Toast) title (medium size): `text-base font-bold` — 16px/700
- Alert (Toast) message (medium size): `text-sm` / weight inherited as 400
- Button labels use Poppins normal 400 at the component's configured size

## Interaction Contracts

### Confirm promise flow

| Step | Trigger | Result |
|------|---------|--------|
| Open | `useConfirm().confirm(options)` called | Returns `Promise<boolean>`, Dialog opens |
| Accept | User clicks accept Button | Promise resolves `true`, Dialog closes |
| Cancel | User clicks cancel Button | Promise resolves `false`, Dialog closes |
| Outside click | User clicks backdrop | Shake animation (100ms), Dialog stays open |
| ESC key | Not handled by this phase | Not in Vue source — no contract |

### Toast lifecycle

| Step | Trigger | Result |
|------|---------|--------|
| Add | `useToast().toast(options)` called | Toast appended to corner queue, entry animation plays |
| Auto-dismiss | `timeout` ms elapsed (default 5000ms) | HIDE action (600ms exit animation), then REMOVE |
| Manual close | User clicks close icon OR `close()` returned from `toast()` | Same HIDE → REMOVE sequence |
| Persistent | `timeout: 0` or `timeout: null` | No auto-dismiss timer set |
