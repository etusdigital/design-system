  <files>src/utils/components/ExpandableContainer.tsx</files>
  <action>
In ExpandableContainer.tsx, the collapsed state (line ~101) applies `h-0 overflow-hidden` simultaneously with `opacity-0 invisible pointer-events-none`. The h-0 clips the element before the opacity transition can animate, making it disappear instantly.

Fix: Remove `h-0 overflow-hidden` from the collapsed CSS classes. The `opacity-0 invisible pointer-events-none` classes combined with the existing `transition: 'opacity 150ms ease, visibility 150ms ease'` inline style are sufficient to hide the content while allowing smooth animation. The `invisible` (visibility:hidden) already prevents interaction and layout contribution when combined with `pointer-events-none`.

Verify the expanded state classes remain: `opacity-100 visible`.
Verify the collapsed state becomes: `opacity-0 invisible pointer-events-none` (NO h-0, NO overflow-hidden).
  </action>

In Select.tsx lines 248-251, when renderOption is not provided, the fallback uses `children` (a static single node with no option/index context) instead of the option's label. This makes options display incorrectly.

Fix: Replace the children fallback in the option mapping with `<span>{getLabel(option)}</span>`. The option mapping code should look like:

```

  : <span>{getLabel(option)}</span>
```

Remove any fallback to `children` in the option rendering path. The `children` prop should only be used for other purposes (like custom trigger content), not as option item fallback.
