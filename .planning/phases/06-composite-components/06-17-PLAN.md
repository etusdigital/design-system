Two bugs to fix in Accordion.tsx:

**Bug 1 — Stale ResizeObserver closure (lines 50-72):**
The mount-only `useEffect` (empty deps) creates a ResizeObserver that calls `resize()`. But `resize()` captures `isExpanded` from the initial render (always `false`). When user clicks to expand, the observer fires and immediately sets maxHeight to 0px because `isExpanded` is stale.

Fix: Use a ref to track the current `isExpanded` value. Add `const isExpandedRef = useRef(isExpanded)` and keep it synced: `isExpandedRef.current = isExpanded` on every render. In the `resize()` function, read from `isExpandedRef.current` instead of `isExpanded` directly. This follows the established pattern from STATE.md decision: "isExpandedRef mutable ref pattern for observer callbacks avoids stale closure without re-registering observers".

Specifically:
1. Add `const isExpandedRef = useRef(isExpanded);` after the existing refs
2. Add `isExpandedRef.current = isExpanded;` before the `resize()` function
3. Change `resize()` to read `isExpandedRef.current` instead of `isExpanded`:
   ```typescript
   function resize() {
     if (!contentRef.current) return;
     contentRef.current.style.maxHeight = isExpandedRef.current
       ? `${contentRef.current.scrollHeight}px`
       : '0px';
   }
   ```

**Bug 2 — Phantom gap below header (line 93):**
The card wrapper div has `gap-sm` class which creates 12px space between header and content even when content is collapsed (maxHeight: 0px still occupies gap space in flex layout).

Fix: Remove `gap-sm` from the card wrapper div. Change line 93 from:
```
<div ref={cardRef} className="w-full flex flex-col gap-sm">
```
to:
```
<div ref={cardRef} className="w-full flex flex-col">
```
The content's own padding/margin handles spacing when expanded.

**Tab icon support (UAT test 4):**

1. In Tab.tsx, add `import { Icon } from '../Icon/Icon';` at the top.

2. Add `isIcon?: boolean` to `TabProps` interface. This enables icon-only mode where each option string is treated as a Material Symbols icon name.

3. Update the options type to support objects with icon field:
   ```typescript
   options?: Array<string | { label: string; icon?: string; [key: string]: any }>;
   ```

4. Replace the simple `getLabel()` call in the button render with two-branch rendering:
   - If option is an object with an `icon` field: render `<Icon name={option.icon} />` followed by `<span>{getLabel(option)}</span>` (icon + text)
   - If option is a string AND `isIcon` is true: render `<Icon name={option} />` only (icon-only mode)
   - Otherwise: render `<span>{getLabel(option)}</span>` (text only, current behavior)

   The button content should be:
   ```tsx
   {typeof option === 'object' && option.icon ? (
     <>
       <Icon name={option.icon} className={styles.icon} />
       <span>{getLabel(option)}</span>
     </>
   ) : isIcon && typeof option === 'string' ? (
     <Icon name={option} className={styles.icon} />
   ) : (
     <span>{getLabel(option)}</span>
   )}
   ```
