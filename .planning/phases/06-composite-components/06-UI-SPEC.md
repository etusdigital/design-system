## Spacing Scale

All spacing values are defined as CSS custom properties and mapped in `tailwind.config.cjs`. Use Tailwind spacing tokens exclusively. All spacing tokens resolve to multiples of 4px.

| Token | CSS Variable | Resolved Value | Usage |
|-------|-------------|----------------|-------|
| xxs | `--spacing-xxs` | 4px | Icon gaps, inline padding, small element gaps |
| xs | `--spacing-xs` | 8px | Compact element spacing, scrollbar dimensions |
| sm | `--spacing-sm` | 12px | Default inner padding for interactive items, card padding |
| base | `--spacing-base` | 16px | Standard element spacing, accordion padding |
| lg | `--spacing-lg` | 20px | Medium component padding |
| xl | `--spacing-xl` | 24px | Section padding, overlay max-width offset |
| 2xl | `--spacing-2xl` | 32px | Layout gaps between sections |
| 3xl | `--spacing-3xl` | 48px | Major section breaks |
| 4xl | `--spacing-4xl` | 56px | Page-level spacing |
| 5xl | `--spacing-5xl` | 64px | Large page sections |

### Border-Width Tokens (not spacing)

The following token resolves to a sub-4px value and is used exclusively as a border width. It is documented here separately and must NOT be used for spacing, padding, or margin.

| Token | CSS Variable | Resolved Value | Usage |
|-------|-------------|----------------|-------|
| — | `--spacing-2xxs` | 2px | Border widths only (e.g. `border-xxs` utility — Drawer border, Accordion card border, SelectContainer border, Calendar day cell selected border, RoundMenu item border) |

Exceptions:
- Drawer/Dialog max-width/max-height: `calc(100% - var(--spacing-xl))` — overlay safety margin
- Touch targets for interactive controls (Stepper circles, RoundMenu buttons, Pagination page buttons): minimum 44px visual touch target, achieved via `py-xs px-xs` (8px each) + content height ≥ 28px

Source: `src/assets/main.css` spacing tokens.

## Typography

All sizes, weights, and line heights reference CSS custom properties from `main.css`. Use the Tailwind utility classes defined in `tailwind.config.cjs`.

| Role | Tailwind Class | Size | Weight | Line Height | Usage |
|------|---------------|------|--------|-------------|-------|
| Label / Caption | `text-xs font-normal` (`.caption-c1`) | 12px | 400 (normal) | — | Option labels, tag text, tab labels, pagination numbers, step labels, ColorPicker type toggles |
| Body / Default | `text-sm font-normal` (`.p3`) | 14px | 400 (normal) | 120% (`leading-lg`) | Dropdown option text, accordion body, filter items, navbar items |
| Sub-heading | `text-base font-semibold` | 16px | 600 (semibold) | 105% (`leading-xs`) | Accordion header text, drawer section headings, dialog titles |
| Heading | `text-xl font-semibold` | 20px | 600 (semibold) | 110% (`leading-sm`) | Navbar title, DatePicker heading, Calendar month/year display |

**Declared font weights: 400 (normal) and 600 (semibold) only.** Medium (500) is absorbed into 400; bold (700) is replaced by 600.

Notes:
- `.p3` (`text-sm font-normal leading-lg`) is the dominant body text class for composites — already in safelist
- `.caption-c1` (`text-xs font-normal`) is used for tab labels, page numbers, and compact interactive items
- Color type indicators in ColorPicker use `text-xs font-normal` for the input type toggles
- Material Symbols icon size within composites: `text-lg` (18px) for chevrons/arrows, `text-xl` (20px) for action icons, `text-2xl` (24px) for navigation

Source: `tailwind.config.cjs` plugin section, `src/assets/main.css` base styles.

## Color

All values reference CSS custom properties. Two themes exist: default (green primary) and `.brius-theme` (blue primary). Components use semantic token names — never hardcoded hex.

| Role | CSS Token | Default Hex | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | `--neutral-background-default` | `#fbfcfd` | Page backgrounds, overlay backdrops behind Drawer/Dialog |
| Secondary (30%) | `--neutral-surface-default` | `#ffffff` | All overlay panels: Drawer, Dialog, Dropdown cards, Select dropdown, DatePicker panel, Accordion card |
| Surface highlight | `--neutral-surface-highlight` | `#f4f6f7` | Tab container background (`notCard=false`), inactive Stepper connector lines, Filter category backgrounds |
| Accent / Primary (10%) | `--primary-interaction-default` | `#329771` | Active tab indicator, active Stepper circle, selected option highlight, DatePicker active date cell, Calendar selected day |
| Foreground high | `--neutral-foreground-high` | `#1b1f22` | Active tab text, Accordion header text, selected option label |
| Foreground low | `--neutral-foreground-low` | `#5f717b` | Default tab text, unselected option text, Pagination inactive pages, Stepper skipped state |
| Foreground disabled | `--neutral-foreground-disabled` | `#7a8d96` | Disabled Select/Dropdown text, disabled Stepper steps |
| Border default | `--neutral-border-default` | `#e3e7ea` | Drawer border, Accordion card border (`border-xxs`), Dropdown card border, SelectContainer border |
| Destructive | `--danger-interaction-default` | `#f03232` | Danger/error state in DatePicker (invalid range), error borders on Select, confirmation destructive actions |

Accent reserved for:
- Active/selected tab button (background + text color change)
- Active Stepper step circle (fill color)
- Selected option in Select/AutoComplete/TagSelect/Dropdown (text color `primary-interaction-selected`)
- Selected Calendar day (background `primary-surface-default`, border `primary-border-default`)
- DatePicker selected date range highlight (`primary-surface-hover`)
- RoundMenu active/focused item
- Filter applied badge count

Source: `src/assets/main.css` color tokens, `src/components/Tab/Tab.vue` `.active-tab`, `src/components/Drawer/Drawer.vue`.

## Component-Specific Visual Contracts

### Overlay Components (Drawer, Dialog — verify only)

- **z-index hierarchy**: Dialog = `z-[1003]`, Toast = `z-[1100]`, Drawer = `z-[1001]`, Dropdown/Select/DatePicker overlays = `z-[1002]`
- **Drawer**: `bg-neutral-surface-default`, `border-xxs border-neutral-default`, `rounded-[position]-base` per side
  - Right: `rounded-l-base`; Left: `rounded-r-base`; Top: `rounded-b-base`; Bottom: `rounded-t-base`
  - Slide animation: `translateX(±100%)` for left/right, `translateY(±100%)` for top/bottom, `transition 0.5s ease`
  - Mobile (< 768px): always slides from bottom, full width
- **Dialog** (verify only — already migrated): scale(0)→scale(1) at center, `transition 0.5s ease`, bounce-warning animation on `noOutsideClose` click

### Dropdown Family (Select, AutoComplete, TagSelect, Dropdown)

- **SelectContainer trigger**: `bg-neutral-surface-default`, `border-xxs border-neutral-default`, `rounded-base`, `px-sm py-xs`
  - Error state: `border-danger-default`
  - Disabled: `bg-neutral-surface-disabled`, text `text-neutral-foreground-disabled`
- **SelectContent dropdown panel**: `bg-neutral-surface-default`, `border-xxs border-neutral-default`, `rounded-base`, `shadow-neutral-default`
- **Option item**: `px-sm py-xs`, hover: `bg-neutral-surface-hover`, selected: `text-primary-interaction-selected`
- **Select multiple mode**: Checkbox rendered inside each Option at `text-sm`
- **AutoComplete**: Identical trigger appearance to Select; internal search input unstyled (inherits from Input)
- **TagSelect tags**: render as `StatusBadge` components with delete icon (Material Symbols `close` at `text-xs`)
- **Dropdown private Option**: `text-sm font-normal`, group headers at `text-xs font-medium text-neutral-foreground-low`

### Accordion

- Card wrapper: `px-base py-xs w-full` with `hover:bg-neutral-surface-hover transition-colors duration-300`
- Header row: flex justify-between, `text-base` text
- Chevron icon: `expand_more` Material Symbol, `text-2xl`, `text-neutral-interaction-default font-bold`, rotates 180° when open (`transition-transform duration-300`)
- Content: `transition-[max-height]` with `duration` prop (150ms–1000ms, clamped), `ease-in-out`
- `noShadow` variant: `border-none`

### Tab

- Container: `p-xxs rounded-sm`, `bg-neutral-surface-highlight` (unless `notCard=true`)
- Default tab button: `py-xs px-xs rounded-xs`, text `text-neutral-interaction-default`, `font-bold text-sm`, hover: `text-primary-interaction-hover`
- Active tab button: `shadow-neutral-selected text-neutral-foreground-high bg-neutral-surface-default`

### Stepper (version 2 only — version 1 dropped per CONTEXT.md)

- Circle: `rounded-full`, size medium = 32px, size large = 40px
- Active state: `bg-primary-interaction-default text-neutral-foreground-negative` (white icon/text on green)
- Past state: `bg-primary-surface-highlight` (light green tint)
- Skipped state: `bg-warning-surface-default` (light yellow)
- Default/future state: `bg-neutral-surface-highlight text-neutral-foreground-disabled`
- Connector line: `bg-neutral-surface-highlight` (inactive), `bg-primary-surface-highlight` (past/active)
- Dynamic background color: inject `--stepper-bg` CSS variable via inline style on root element; connector lines use `var(--stepper-bg)` to blend with page background
- No `version` prop — removed per CONTEXT.md decision

### Calendar

- Month grid container: `bg-neutral-surface-default`, `rounded-base`
- Day cell — unselected: `text-sm text-neutral-foreground-high`, hover: `bg-neutral-surface-hover rounded-xs`
- Day cell — selected (date mode): `bg-primary-surface-default text-primary-foreground-high rounded-xs border-xxs border-primary-default`
- Day cell — range (period mode): `bg-primary-surface-hover` between start/end; endpoints get `bg-primary-interaction-default text-neutral-foreground-negative`
- Day cell — today: `font-semibold`
- Day cell — outside current month: `text-neutral-foreground-disabled`
- Day cell — disabled: `text-neutral-foreground-disabled cursor-not-allowed`
- Slide-fade transition on month change: `useTransition` + CSS classes `slide-fade-left` / `slide-fade-right`, `transition all 0.3s ease`
- Navigation arrows: Material Symbols `chevron_left` / `chevron_right` at `text-lg`, `text-neutral-interaction-default`
  - Previous month button: `aria-label="Previous month"`
  - Next month button: `aria-label="Next month"`
- Month/Year picker dialog (`Calendar.DateDialog`): same card styling as SelectContent

### DatePicker

- Trigger wraps Calendar in `ExpandableContainer` — identical trigger appearance to Select
- Action buttons: `clearLabel` / `applyLabel` / `compareLabel` as string props rendered as DS Button components
- Clear button: neutral variant; Apply button: primary variant
- Compare mode: secondary panel visible alongside primary calendar panel

### Drawer (Carousel)

- Track: `overflow-hidden`, no scrollbar visible
- Slide transition: `transform translateX` / `translateY`, `transition 0.3s ease` per slide via CSS
- Autoplay indicator: none (no visual indicator per Vue source)
- Navigation arrows (if enabled): Material Symbols `chevron_left` / `chevron_right`, positioned absolutely, `text-2xl`
  - Previous slide button: `aria-label="Previous slide"`
  - Next slide button: `aria-label="Next slide"`

### Filter

- Category header: `px-sm py-xs`, flex row, expand icon `expand_more`
- Category expand/collapse: `max-height` CSS transition (no `useTransition` — in-place animation per CONTEXT.md)
- Sub-options: Checkbox + label at `text-sm`
- Action row: Clear (`clearLabel`) + Apply (`applyLabel`) as DS Button components — neutral + primary variants

### Pagination

- Page button: `py-xs px-xs rounded-xs text-xs font-medium`
- Active page: `bg-primary-interaction-default text-neutral-foreground-negative`
- Inactive page: `text-neutral-foreground-low hover:bg-neutral-surface-hover`
- Ellipsis (−1 sentinel): rendered as `...` text, `text-neutral-foreground-disabled`, not interactive
- Prev/Next buttons: Material Symbols `chevron_left` / `chevron_right` icons
  - Previous page button: `aria-label="Previous page"`
  - Next page button: `aria-label="Next page"`

### Navbar

- Container: `bg-neutral-surface-default`, `border-b-xxs border-neutral-default`, horizontal flex
- Logo slot: left-anchored
- Navigation items: `text-sm font-normal leading-lg text-primary-foreground-high`, gap `spacing-base`
- Dropdown trigger in Navbar: inherits Dropdown component visual styles
- Profile slot: right-anchored, uses Profile component

### RoundMenu

- Menu items positioned with `cos(angle) / sin(angle)` × radius, `translate3d`
- Item button: `rounded-full`, `bg-neutral-surface-default border-xxs border-neutral-default shadow-neutral-default`
- Active/expanded center trigger: `bg-primary-interaction-default text-neutral-foreground-negative`
- Animation: `transition transform 0.3s ease`, items scale from 0 at center to full radius

### ColorPicker

- Container: `bg-neutral-surface-default rounded-base shadow-neutral-default p-base`
- Color area (gradient canvas): fixed width, `border-radius-base`
- Hue slider + opacity slider: `border-radius-full`, height 12px, cursor circle `w-xs h-xs` (8px) rounded-full with white border
- Type toggle (hexa/rgba/hsla/hsva/hwb): Tab-style small buttons, `text-xs`
- Input `type="color"` integration: hidden native input synced to ColorPicker value via `value` prop

## CSS Conventions (Phase 6 Enforcement)

These patterns are locked by Phases 1–5 and must be followed without deviation:

1. **CSS Module files**: `ComponentName.module.css` for each composite component; plain `.css` for portal-rendered components (Drawer, any additional toast-like overlays) that render outside component tree
2. **`@reference "../../assets/main.css"`**: First line of every CSS file to enable Tailwind `@apply` directives
3. **`className` prop**: Every component accepts and forwards a `className` prop — applied to the root element
4. **`clsx`**: Used for conditional class logic; no string concatenation
5. **CSS variable injection for dynamic values**: `style={{ '--stepper-bg': background }}` on Stepper root; similar pattern for any dynamic color/size value used in CSS pseudo-elements
6. **No `computedStyleMap()`**: Use `getComputedStyle()` for CSS variable reading (jsdom compatibility, established in Phase 3 Slider)
7. **`useLayoutEffect` for DOM measurements**: Carousel slide width reads must use `useLayoutEffect` to prevent visual flicker
