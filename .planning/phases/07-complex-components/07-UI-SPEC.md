## Spacing Scale

The project uses a custom spacing scale backed by CSS variables. The values below are the token-resolved px values from `main.css`.

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xxs | --spacing-xxs | 4px | Icon gaps, tight inline padding |
| xs | --spacing-xs | 8px | Compact element spacing (table cell padding, tree indent step) |
| sm | --spacing-sm | 12px | Medium compact spacing |
| base | --spacing-base | 16px | Default element spacing (toolbar buttons, sidebar option padding) |
| lg | --spacing-lg | 20px | Component internal section spacing |
| xl | --spacing-xl | 24px | Section padding (table header, sidebar group headers) |
| 2xl | --spacing-2xl | 32px | Layout gaps |
| 3xl | --spacing-3xl | 48px | Major section breaks |
| 5xl | --spacing-5xl | 64px | Page-level spacing |

Exceptions:
- Table cell touch targets: minimum 44px row height to meet accessibility tap target requirements
- Tree node indent: each nesting level adds `--spacing-base` (16px) of left padding per depth level
- Sidebar collapsed width: fixed at `--spacing-5xl` (64px) icon-only rail; expanded to fixed width matching Vue source

Source: `src/assets/main.css` spacing tokens

## Typography

All type roles use Poppins (`var(--font-sans)`). Values resolved from `main.css` CSS variables and the Tailwind component utilities (`.p1`–`.p4`, `.caption-c1`, `.caption-c2`) declared in `tailwind.config.cjs`.

| Role | Size Token | Resolved Size | Weight Token | Resolved Weight | Line Height Token | Resolved LH |
|------|-----------|---------------|-------------|-----------------|------------------|-------------|
| Body / table cell | text-sm (.p3) | 14px | font-normal | 400 | leading-lg | 120% |
| Label / column header | text-xs (.caption-c1) | 12px | font-medium | 500 | — (default) | 120% |
| Heading / section title | text-base (.p2) | 16px | font-normal | 400 | leading-lg | 120% |
| Caption / helper text | text-xxs (.caption-c2) | 10px | font-normal | 400 | — (default) | 120% |

Phase-specific notes:
- **Table column headers**: `text-xs font-medium` (caption-c1 utility) — matches Vue `<th>` styling
- **Table cell body**: `text-sm font-normal` (p3 utility)
- **RichTextEditor toolbar**: icon buttons use 20px Material Symbols icons (size=20 on Icon component); font-size Select uses compact variant of DS Select
- **Sidebar nav items**: `text-sm font-normal` for option labels; active item uses `font-medium`
- **History labels**: `text-xs font-medium` for step labels; `text-xxs font-normal` for sub-labels
- **Tree node labels**: `text-sm font-normal`; selected node label uses `font-medium`

Source: `tailwind.config.cjs` component utilities + `main.css` font tokens

## Color

All color values reference CSS custom properties. Hex values below are the resolved defaults from `main.css` `:root`.

| Role | CSS Variable | Hex | Usage |
|------|-------------|-----|-------|
| Dominant (60%) | --neutral-background-default | #fbfcfd | Page background, table row default background |
| Secondary (30%) | --neutral-surface-default / --neutral-surface-highlight | #ffffff / #f4f6f7 | Cards, sidebar background, table header background |
| Accent (10%) | --primary-interaction-default | #329771 | Active sidebar item indicator, selected tree node, active History step circle, table row selected checkbox |
| Destructive | --danger-interaction-default | (var(--danger-interaction-default)) | Destructive actions only — no destructive actions in Phase 7 |

Accent reserved for:
- Active/selected sidebar navigation item left border indicator
- Selected tree node label and checkbox fill
- Active History timeline step circle fill
- Table row selection checkbox (checked state)
- Table sort indicator arrow (active sort column)

Semantic color usage for Phase 7 components:

| Component | Semantic colors used |
|-----------|---------------------|
| Table | neutral (rows, header), primary (selected rows), danger (delete action if present) |
| Tree | neutral (nodes), primary (selected node) |
| Sidebar | neutral (default items), primary (active item), neutral-foreground-low (icons) |
| History | primary / info / success / warning / danger / neutral (step type variants, one per step via `type` prop) |
| RichTextEditor | neutral (toolbar, content area), primary (focus border) |
| Crop | neutral (overlay, controls) |

Source: `main.css` color tokens, `tailwind.config.cjs` color aliases

## Copywriting Contract

Phase 7 components are developer-facing library components, not end-user applications. Copywriting is therefore component-internal default copy that consumers may override via props.

| Element | Copy |
|---------|------|
| Table empty state heading | "No items found" |
| Table empty state body | "There are no items to display. Add data to see it here." |
| Table loading state | "Loading..." (shown as Skeleton rows, no text label) |
| Table items-per-page label | "Items per page" |
| Table pagination summary | "{min}–{max} of {total} items" |
| Table select-all label | (checkbox only — no text label, aria-label="Select all rows") |
| Tree empty state | "No options available." |
| Sidebar no selection | (no copy — sidebar opens to first item active by default) |
| History no options | (component renders nothing — empty options array renders empty container) |
| RichTextEditor placeholder | "Start typing..." |
| RichTextEditor toolbar — bold | aria-label="Bold" |
| RichTextEditor toolbar — italic | aria-label="Italic" |
| RichTextEditor toolbar — underline | aria-label="Underline" |
| Crop instructions | (no visible text — interaction via drag handles only) |
| Error state (Table fetch fail) | Not in scope — consumer handles error state via `items` prop |
| Destructive confirmation | No destructive actions in Phase 7 components |

Source: Default — inferred from Vue source component behavior and CONTEXT.md decisions

## Component Interaction States

### Table

| State | Visual Treatment |
|-------|----------------|
| Row default | background: neutral-background-default (#fbfcfd) |
| Row hover (hasHover=true) | background: neutral-surface-hover (--neutral-surface-hover, #f4f6f7) |
| Row selected (enableSelection) | background: primary-surface-default (--primary-surface-default, #effaf4); checkbox checked with primary accent |
| Column sortable — default | sort icon: neutral-foreground-low opacity |
| Column sortable — active asc | sort icon: primary-interaction-default, arrow up |
| Column sortable — active desc | sort icon: primary-interaction-default, arrow down |
| Loading | Skeleton rows (DS Skeleton component) replace data rows; header remains visible |
| Empty | Table.EmptyState slot content centered in tbody area |
| Header fixed (isHeaderFixed) | thead: position sticky; top: 0; z-index: 1; background: neutral-surface-default |

### Tree

| State | Visual Treatment |
|-------|----------------|
| Node default | text: neutral-foreground-high; icon: neutral-foreground-low |
| Node hover | background: neutral-surface-hover |
| Node selected | text: primary-foreground-low; background: primary-surface-default; left border: 2px primary-interaction-default |
| Node expand/collapse | Icon rotates 90deg (expand_more → rotate 0deg collapsed, -90deg expanded); no animation beyond CSS transition |
| Node disabled | text: neutral-foreground-disabled; pointer-events: none |
| Nested indent | Left padding += --spacing-base (16px) per depth level |

### Sidebar

| State | Visual Treatment |
|-------|----------------|
| Option default | text: neutral-foreground-high; icon: neutral-foreground-low |
| Option hover | background: neutral-surface-hover |
| Option active | left border: 3px solid primary-interaction-default; text: primary-foreground-low; background: primary-surface-default |
| Sub-option active | left border: 3px solid primary-interaction-default; text: primary-foreground-low |
| Collapsed (icon-only) | Width: --spacing-5xl (64px); labels hidden; icons centered |
| Expanded | Width: matches Vue source layout; labels visible |
| Height | calc(100vh - navbarHeight) via querySelector('.navbar') |

### History

| Step disabled | circle: neutral-surface-disabled; pointer-events: none |
| Connector line | background: neutral-border-default; active connector: primary-interaction-default |
| Position top/bottom | flex-row layout; connectors are horizontal |
