<task type="auto">
  <name>Task 1: Fix History CSS — circle sizes, connector thickness, default type fills</name>
  <files>src/components/History/History.module.css</files>
  <action>
Three CSS fixes in History.module.css:

1. **Override .circleIcon dimensions** — The `.circleIcon` class (line ~193) inherits the tiny `.circle` base of `w-[10px] h-[10px]`. Add explicit size overrides to `.circleIcon`:
   ```
   .circleIcon {
     @apply flex items-center justify-center p-sm text-neutral-foreground-negative bg-neutral-surface-disabled
            w-[32px] h-[32px] min-w-[32px] min-h-[32px] rounded-full;
   }
   ```
   This makes icon circles 32x32 (large colored circles) instead of inheriting 10x10.

2. **Increase connector border thickness** — The `.dataList` rules for each position use `border-l-[1.5px]` (right position, line ~223), `border-r-[1.5px]` (left position, line ~261), `border-t-[1.5px]` (top position, line ~299), `border-[1.5px]` (bottom position, line ~340). Change all to `3px` equivalents:
   - `.option.right .dataList`: `border-l-[3px]`
   - `.option.left .dataList`: `border-r-[3px]`
   - `.option.top .dataList`: `border-t-[3px]`
   - `.option.bottom .dataList`: `border-[3px]`

3. **Add default type-colored circle fill rules** — Currently, type-colored circle fills (e.g. `.primary.active .circle { bg-primary-interaction-default }`) only apply with `.active` class. Add NON-active type-colored circle fill rules for each type AFTER the existing type hover rules (after line ~56) and BEFORE the active variants section:
   ```
   /* Default type circle fills (idle, non-active state) */
   .primary .circle { @apply bg-primary-interaction-default; }
   .info .circle { @apply bg-informative-interaction-default; }
   .success .circle { @apply bg-success-interaction-default; }
   .warning .circle { @apply bg-warning-interaction-default; }
   .danger .circle { @apply bg-danger-interaction-default; }
   .neutral .circle { @apply bg-neutral-interaction-default; }
   ```
   The existing `.active` rules already set z-[2] which is fine. The disabled rules (lines 120-178) also already set their own circle colors which will take specificity precedence.

Two independent fixes:

**Crop.tsx — Unique SVG mask ID per instance:**
1. Import `useId` from React: add to the existing React import at the top of the file.
2. Inside the Crop component function, call `const maskId = useId();` to generate a unique ID per instance.
3. Replace the hardcoded `id="highlight-mask"` on the `<mask>` element (line ~210) with `id={maskId}`.
4. Replace the hardcoded `mask="url(#highlight-mask)"` on the overlay `<rect>` (line ~228) with `` mask={`url(#${maskId})`} ``.
5. Also scope the window-level drag event listeners to the component container. In the existing mousedown/mousemove/mouseup handlers that attach to `window`, change them to attach to `containerRef.current` (or the component's root element ref) instead. If the component already uses a container ref, use it. If not, the window listeners may be acceptable if the component properly cleans up — prioritize the mask ID fix as the primary cause of cross-instance interference.

**RichTextEditor.tsx — Sanitizer allowlist:**
1. On line 31, add `'font'` to the ALLOWED_TAGS array:
