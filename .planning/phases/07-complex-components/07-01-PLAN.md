**Props interface:**
```typescript
interface HistoryOption {
  [key: string]: any;
}

interface HistoryProps {
  value?: any;
  defaultValue?: any;
  onChange?: (value: any) => void;
  options: HistoryOption[];
  position?: 'top' | 'bottom' | 'left' | 'right';
  type?: 'primary' | 'info' | 'success' | 'warning' | 'danger' | 'neutral';
  disabled?: boolean;
  renderOption?: (option: any, index: number, active: boolean) => React.ReactNode;
  className?: string;
}
```

**Key implementation details:**
1. Use `useControllable` for value binding: `const [model, setModel] = useControllable({ value, defaultValue: null, onChange })`
2. Active check: `const isActive = (option: any, index: number) => (index === 0 && !model && !disabled) || option === model`
3. Position determines flex direction: `top`/`bottom` = `flex-row`, `left`/`right` = `flex-column`
4. Each option renders: circle indicator + connector line + content area
5. Circle fill color uses type prop CSS variable: `--{type}-interaction-default` for active steps
6. Connector line between steps: horizontal for top/bottom, vertical for left/right
7. Active connector: uses type color; inactive: neutral-border-default
8. Vue slot `#option` maps to `renderOption` render prop: `renderOption?.(option, index, isActive(option, index))`
9. Port ALL CSS from History.vue `<style>` block into History.module.css
10. Use clsx for conditional class composition: `clsx(styles.history, styles[position], className)`
11. Each step type variant gets a CSS class: `styles.primary`, `styles.info`, etc. that sets the `--step-color` CSS variable

**CSS Module classes needed:**
- `.history` — root container
- `.top`, `.bottom`, `.left`, `.right` — position variants controlling flex direction
- `.option` — individual step wrapper
- `.circle` — step indicator dot
- `.circleActive` — active step circle with type color fill
- `.connector` — line between steps
- `.connectorActive` — active connector with type color
- `.content` — step content area
- `.primary`, `.info`, `.success`, `.warning`, `.danger`, `.neutral` — type color variants

**Props interface:**
```typescript
interface CropProps {
  src?: string;
  zoom?: number;
  defaultZoom?: number;
  onZoomChange?: (zoom: number) => void;
  onChange?: (croppedDataUrl: string) => void;
  disabled?: boolean;
  className?: string;
}
```

**Key implementation details:**
1. DOM refs (mutable, NOT state): `isDraggingRef`, `startXRef`, `startYRef`, `translateXRef`, `translateYRef` for drag tracking
2. State for rendering: `const [svgDims, setSvgDims] = useState({ width: 0, height: 0 })` for SVG viewBox
3. State for zoom: `const [zoom, setZoom] = useControllable({ value: props.zoom, defaultValue: props.defaultZoom ?? 1, onChange: props.onZoomChange })`
4. ResizeObserver in `useEffect([])`: observe bCropRef for container size changes, update svgDims
5. Window event listeners in same `useEffect([])`: mousemove (updateCropArea), mouseup (stop drag), touchmove, touchend
6. Clean up all listeners in useEffect return
7. `getPosition` imported from `src/utils/index.ts` for clamping drag position
8. SVG mask: `<svg>` with `<defs><mask>` containing a `<rect ref={selectedAreaRef}>` — attributes updated imperatively during drag via `setAttribute`
9. Canvas crop operation: create offscreen canvas, draw image with zoom/translate, call `canvas.toDataURL()`, emit via onChange
10. Use `getComputedStyle(element).getPropertyValue('--border-radius-xl')` for SVG mask rx/ry — NOT computedStyleMap
11. Image `onLoad`: set parsedImgSrc state, compute initial SVG dimensions
12. Touch event support: touchstart/touchmove/touchend mirror mouse events
13. Zoom slider: `<input type="range">` with min/max/step, updates zoom state
